# Асинхронность в JavaScript: Полное руководство

Асинхронность позволяет выполнять длительные операции (запросы к серверу, чтение файлов, таймеры) не блокируя основной поток выполнения программы. Это критически важно для создания отзывчивых интерфейсов.

---

## 1. Что такое асинхронность?

В синхронном коде каждая операция ждет завершения предыдущей. Если одна задача занимает 10 секунд, вся программа "зависает".
Асинхронный код позволяет "запустить" задачу и продолжить выполнение остального кода, получив результат позже.

---

## 2. Промисы (Promises)

Промис — это объект, представляющий результат асинхронной операции, который будет доступен в будущем.

У промиса есть три состояния:
1. **Pending** (Ожидание) — начальное состояние.
2. **Fulfilled** (Выполнено успешно) — операция завершена успешно.
3. **Rejected** (Ошибка) — операция завершена с ошибкой.

```javascript
const myPromise = new Promise((resolve, reject) => {
  const success = true;
  if (success) {
    resolve("Данные получены!");
  } else {
    reject("Ошибка сети");
  }
});

myPromise
  .then(data => console.log(data))    // Обработка успеха
  .catch(err => console.error(err))  // Обработка ошибки
  .finally(() => console.log("Завершено")); // Выполнится всегда
```

---

## 3. Async / Await

Это современный и наиболее читаемый способ работы с промисами, который делает асинхронный код похожим на синхронный.

- `async` перед функцией означает, что она всегда возвращает промис.
- `await` можно использовать только внутри `async`-функции. Он заставляет JS ждать завершения промиса.

```javascript
async function fetchData() {
  try {
    const response = await fetch("https://api.example.com/data");
    const data = await response.json();
    console.log(data);
  } catch (error) {
    console.error("Произошла ошибка:", error);
  }
}
```

## 4. Таймеры: `setTimeout` и `setInterval`

- `setTimeout(callback, delay)` — выполняет функцию один раз через указанное время (в мс).
- `setInterval(callback, delay)` — выполняет функцию регулярно через равные промежутки времени.

```javascript
setTimeout(() => console.log("Прошла 1 секунда"), 1000);
```

---

## 5. Основы Event Loop (Цикл событий)

JavaScript — однопоточный язык. Чтобы справляться с асинхронностью, он использует Event Loop.

1. **Call Stack (Стек вызовов)**: сюда попадает синхронный код.
2. **Web APIs**: здесь выполняются асинхронные задачи (таймеры, запросы).
3. **Callback Queue (Очередь задач)**: сюда попадают результаты задач из Web APIs.
4. **Event Loop**: если стек вызовов пуст, он переносит первую задачу из очереди в стек.

---

## 6. Микрозадачи и Макрозадачи

Задачи в очереди имеют разный приоритет:
- **Микрозадачи**: промисы (`.then/catch/finally`). Выполняются сразу после текущего кода, до того как браузер перерисует страницу.
- **Макрозадачи**: `setTimeout`, `setInterval`, события браузера. Выполняются после микрозадач.

---

## Лучшие практики

1. **Всегда используйте `try...catch`** с `async/await`, чтобы избежать необработанных ошибок.
2. **Избегайте "Callback Hell"** (вложенных колбэков). Используйте промисы или `async/await`.
3. **Используйте `Promise.all()`**, если вам нужно запустить несколько независимых запросов одновременно.
4. **Помните о контексте**. Асинхронные функции всегда возвращают промис, даже если вы возвращаете обычное значение.
