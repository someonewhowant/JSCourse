# Классы и ООП в JavaScript: Полное руководство

Объектно-ориентированное программирование (ООП) — это парадигма программирования, основанная на концепции "объектов", которые могут содержать данные и код. В JavaScript классы являются "синтаксическим сахаром" над прототипным наследованием, делая работу с объектами более привычной для разработчиков на других языках.

---

## 1. Синтаксис классов

Класс — это шаблон для создания объектов.

```javascript
class User {
  // Конструктор вызывается автоматически при создании экземпляра через 'new'
  constructor(name, role) {
    this.name = name;
    this.role = role;
  }

  // Метод класса (записывается в прототип)
  sayHi() {
    console.log(`Привет, я ${this.name}, мой статус: ${this.role}`);
  }
}

const ivan = new User("Иван", "Админ");
ivan.sayHi(); // "Привет, я Иван, мой статус: Админ"
```

---

## 2. Наследование (extends)

Классы могут наследоваться друг от друга, перенимая свойства и методы родителя.

```javascript
class Animal {
  constructor(name) {
    this.name = name;
  }
  eat() {
    console.log(`${this.name} ест...`);
  }
}

class Rabbit extends Animal {
  // Переопределение метода
  eat() {
    super.eat(); // Вызов родительского метода через super
    console.log(`${this.name} грызет морковку`);
  }
}

const bugz = new Rabbit("Багз");
bugz.eat();
```

## 3. Геттеры и Сеттеры

Позволяют контролировать доступ к свойствам и выполнять логику при чтении или записи.

```javascript
class Rect {
  constructor(width, height) {
    this.width = width;
    this.height = height;
  }

  get area() {
    return this.width * this.height;
  }

  set width(value) {
    if (value <= 0) throw new Error("Ширина должна быть больше нуля");
    this._width = value; // Используем _ для предотвращения рекурсии
  }
}
```

## 4. Статические методы и свойства

Они принадлежат самому классу, а не его экземплярам.

```javascript
class Point {
  static origin = new Point(0, 0);

  constructor(x, y) {
    this.x = x;
    this.y = y;
  }

  static distance(a, b) {
    const dx = a.x - b.x;
    const dy = a.y - b.y;
    return Math.hypot(dx, dy);
  }
}

const p1 = new Point(5, 5);
const p2 = new Point(10, 10);
console.log(Point.distance(p1, p2)); // Вызываем у класса, а не у p1
```

## 5. Приватные свойства (`#`)

Современный JavaScript поддерживает по-настоящему приватные поля, которые недоступны вне класса.

```javascript
class BankAccount {
  #balance = 0; // Приватное поле

  deposit(amount) {
    this.#balance += amount;
  }

  getBalance() {
    return this.#balance;
  }
}
```

## 6. Принципы ООП (кратко)

1.  **Инкапсуляция**: Скрытие внутренней реализации и предоставление интерфейса (например, приватные поля).
2.  **Наследование**: Создание новых классов на базе существующих (`extends`).
3.  **Полиморфизм**: Возможность объектов с разным типом иметь одинаковые имена методов, которые ведут себя по-разному.
4.  **Абстракция**: Выделение только необходимых характеристик объекта, игнорируя детали.

---

## Лучшие практики

1.  **Начинайте имена классов с большой буквы** (PascalCase): `UserAccount`, `DataProcessor`.
2.  **Используйте `super()`** в конструкторе потомка перед обращением к `this`.
3.  **Не делайте классы слишком большими**. Если класс делает всё — это плохой признак (нарушение принципа единственной ответственности).
4.  **Предпочитайте композицию наследованию**, если связь между объектами не является строгой иерархией ("является" vs "содержит").
