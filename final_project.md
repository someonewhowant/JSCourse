# Финальный проект: Умный список задач (Smart Task Tracker)

Поздравляем! Вы прошли долгий путь от переменных до архитектуры приложений. Пришло время применить все ваши знания на практике. Ваша задача — создать современное приложение для управления задачами.

---

## Техническое задание (ТЗ)

Приложение должно представлять собой интерактивный список задач с сохранением данных и интеграцией с внешним API.

### Основные функции:
1.  **Управление задачами**:
    - Добавление новой задачи через форму.
    - Удаление задачи.
    - Отметка задачи как "выполненной" (изменение стиля).
2.  **Хранение данных**:
    - Использование `localStorage`, чтобы список задач не пропадал после перезагрузки страницы.
3.  **Интеграция с API**:
    - Добавьте блок "Погода", который запрашивает данные о текущей погоде в вашем городе через `fetch` (можно использовать бесплатный API, например, OpenWeatherMap).
4.  **Фильтрация**:
    - Возможность просматривать "Все", "Только выполненные" или "Только активные" задачи.

---

## Требования к коду:

1.  **Архитектура**:
    - Используйте **классы** для описания логики задачи и всего приложения.
    - Разделите код на **модули** (минимум 3 файла: `main.js`, `Task.js`, `Storage.js`).
2.  **Качество кода**:
    - Обработка ошибок при работе с сетью (`try...catch`).
    - Понятные имена переменных и функций.
    - Отсутствие "магических чисел" и дублирования кода.
3.  **Интерфейс (DOM)**:
    - Использование `template` или динамическое создание элементов через `createElement`.
    - Делегирование событий для обработки кликов по кнопкам удаления.

---

## Рекомендуемая структура проекта:

```text
/project
  ├── index.html
  ├── style.css
  └── js/
      ├── main.js       # Точка входа, инициализация
      ├── Task.js       # Класс одной задачи
      ├── TaskList.js   # Класс для управления списком (логика)
      ├── Storage.js    # Работа с LocalStorage
      └── Weather.js    # Работа с Weather API
```

---

## С чего начать (Подсказки):

1.  **HTML/CSS**: Создайте базовую разметку. Не тратьте слишком много времени на дизайн, главное — функционал.
2.  **Класс Task**: Опишите, какие данные есть у задачи (id, текст, статус).
3.  **Рендеринг**: Напишите функцию, которая отрисовывает массив задач в DOM.
4.  **События**: Настройте обработчик `submit` на форме добавления.
5.  **LocalStorage**: Напишите методы `save` и `load` в отдельном модуле, чтобы синхронизировать массив задач с хранилищем.
6.  **Погода**: Напишите асинхронную функцию, которая делает `fetch` и выводит температуру в отдельный блок.
