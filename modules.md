# ES-модули в JavaScript: Полное руководство

Модули позволяют разбивать код на несколько файлов. Это делает проект более организованным, облегчает поддержку и позволяет повторно использовать код в разных частях приложения.

---

## 1. Что такое модуль?

Модуль — это просто файл с кодом. В модулях JavaScript по умолчанию включен строгий режим (`use strict`). Переменные, объявленные в модуле, не видны в других файлах, пока вы их явно не экспортируете.

---

## 2. Экспорт (export)

Чтобы сделать переменную, функцию или класс доступными для других файлов, используется ключевое слово `export`.

### Именованный экспорт
Вы можете экспортировать несколько сущностей из одного файла.

```javascript
// file: math.js
export const PI = 3.14;

export function add(a, b) {
  return a + b;
}
```

### Экспорт по умолчанию (export default)
В файле может быть только один экспорт по умолчанию. Обычно это основной объект или класс файла.

```javascript
// file: User.js
export default class User {
  constructor(name) {
    this.name = name;
  }
}
```

---

## 3. Импорт (import)

Для использования кода из другого модуля используется ключевое слово `import`.

### Импорт именованных сущностей
```javascript
import { PI, add } from './math.js';

console.log(add(PI, 2));
```

### Импорт экспорта по умолчанию
При импорте сущности по умолчанию вы можете дать ей любое имя.
```javascript
import UserAccount from './User.js';
const user = new UserAccount("Иван");
```

### Импорт всего содержимого как объекта
```javascript
import * as math from './math.js';
console.log(math.PI);
```

---

## 4. Переименование при импорте и экспорте (as)

Если имена в разных модулях совпадают, их можно переименовать.

```javascript
import { add as sum } from './math.js';
console.log(sum(1, 2));
```

---

## 5. Динамический импорт

Иногда нужно загрузить модуль только при определенном условии. Для этого используется функция `import()`, которая возвращает промис.

```javascript
if (userIsAdmin) {
  const module = await import('./adminPanel.js');
  module.show();
}
```

---

## 6. Модули в браузере

Чтобы браузер понял, что файл является модулем, нужно добавить атрибут `type="module"` в тег `<script>`.

```html
<script type="module" src="main.js"></script>
```

---

## Лучшие практики

1. **Один файл — одна логическая единица**. Не сваливайте всё в один `utils.js`.
2. **Используйте именованный экспорт**, если в файле много мелких функций. Это помогает инструментам разработки (IDE) лучше подсказывать имена.
3. **Избегайте циклической зависимости** (когда файл A импортирует B, а B импортирует A).
4. **Не забывайте расширение `.js`** при импорте в браузере или современных версиях Node.js без сборщиков.
