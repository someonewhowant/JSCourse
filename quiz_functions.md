### Вопрос 1
В чем основное отличие Function Declaration от Function Expression?

A) Function Expression можно вызывать до объявления  
B) Function Declaration всплывает (можно вызвать до объявления), а Function Expression — нет  
C) Между ними нет никакой разницы  
D) Function Declaration работает быстрее

### Вопрос 3
Какой синтаксис у стрелочной функции, которая просто возвращает сумму двух чисел?

A) `const sum = (a, b) => { a + b }`  
B) `const sum = (a, b) => return a + b`  
C) `const sum = (a, b) => a + b`  
D) `function sum(a, b) => a + b`

### Вопрос 4
Что такое "остаточные параметры" (...rest)?

A) Способ передать массив как список аргументов  
B) Синтаксис, позволяющий собрать все лишние аргументы в массив  
C) Параметры, которые всегда равны `undefined`  
D) Список глобальных переменных  

---

### Вопрос 5
Какое имя функции лучше всего подходит согласно стандартам чистого кода?

A) `userData()`  
B) `calculation()`  
C) `getUserData()`  
D) `process()`  

---

## Ответы

<details>
<summary>Нажмите, чтобы увидеть ответы</summary>

1. **B) Function Declaration всплывает...** (Это позволяет организовывать код более гибко).
2. **C) undefined** (Это значение по умолчанию для всех функций без явного возврата).
3. **C) const sum = (a, b) => a + b** (При однострочной записи `return` и фигурные скобки подразумеваются).
4. **B) Синтаксис, позволяющий собрать все лишние аргументы в массив.**
5. **C) getUserData()** (Имя функции должно быть глаголом, описывающим конкретное действие).

</details>
