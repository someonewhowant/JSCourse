### Вопрос 1
Какое значение будет выведено в консоль при выполнении следующего кода?

```javascript
console.log(a);
var a = 10;
```

A) `10`  
B) `ReferenceError`  
C) `undefined`  
D) `null`

### Вопрос 2
В чем основное различие между `let` и `var` с точки зрения области видимости?

A) `var` имеет блочную область видимости, а `let` — функциональную  
B) `let` имеет блочную область видимости, а `var` — функциональную

### Вопрос 3
Что произойдет при попытке выполнить этот код?

```javascript
const user = { name: "Ivan" };
user.name = "Dmitry";
console.log(user.name);
```

A) Ошибка `TypeError`, так как `user` — константа  
B) В консоль выведется `"Ivan"`  
C) В консоль выведется `"Dmitry"`  
D) Код не выполнится из-за синтаксической ошибки

### Вопрос 4
Что такое "Временная мертвая зона" (Temporal Dead Zone)?

A) Время, когда скрипт перестает работать из-за ошибки  
B) Состояние переменной `var` до её инициализации  
C) Период от начала блока до момента объявления переменной через `let` или `const`, когда к ней нельзя обратиться  
D) Область памяти, которая автоматически очищается сборщиком мусора  

---

### Вопрос 5
Что выведет этот код (демонстрация замыкания)?

```javascript
function outer() {
  let x = 10;
  return function inner() {
    console.log(x);
  };
}
const closureFn = outer();
closureFn();
```

A) `undefined`  
B) `ReferenceError`  
C) `10`  
D) `null`  

---

## Ответы

<details>
<summary>Нажмите, чтобы увидеть ответы</summary>

1. **C) undefined** (из-за хойстинга `var` объявляется, но не инициализируется).
2. **B) let имеет блочную область видимости, а var — функциональную.**
3. **C) "Dmitry"** (`const` запрещает переназначение самой переменной, но позволяет изменять свойства объекта).
4. **C) Период от начала блока до момента объявления переменной...**
5. **C) 10** (функция `inner` сохраняет доступ к переменной `x` из своей родительской области видимости).

</details>
