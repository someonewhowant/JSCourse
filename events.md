# События в JavaScript: Полное руководство

События — это сигналы от браузера о том, что что-то произошло: пользователь кликнул на кнопку, страница загрузилась, или была нажата клавиша на клавиатуре. Обработка событий позволяет делать страницы интерактивными.

---

## 1. Обработчики событий

Существует несколько способов назначить обработчик, но современным стандартом является `addEventListener`.

### addEventListener
Позволяет назначать несколько обработчиков на одно событие.

```javascript
const btn = document.querySelector('button');

function handleClick() {
  console.log('Клик!');
}

btn.addEventListener('click', handleClick);

// Удаление обработчика (нужна ссылка на функцию)
btn.removeEventListener('click', handleClick);
```

---

## 2. Объект события (event)

При возникновении события браузер создает **объект события**, который передается первым аргументом в функцию-обработчик. В нем содержится подробная информация о происшествии.

```javascript
btn.addEventListener('click', (event) => {
  console.log(event.type); // "click"
  console.log(event.target); // элемент, на котором произошло событие
  console.log(event.clientX, event.clientY); // координаты клика
});
```

---

## 3. Всплытие и погружение

Когда событие происходит на элементе, оно сначала проходит через всех его предков (погружение), достигает цели, а затем поднимается обратно к корню документа (всплытие).

- **Всплытие (Bubbling)**: событие срабатывает на самом элементе, затем на его родителе, затем выше по дереву. Это поведение по умолчанию.
- **event.stopPropagation()**: останавливает всплытие события дальше.

---

## 4. Делегирование событий

Вместо того чтобы назначать обработчик каждому из 100 элементов, мы можем назначить один обработчик их общему предку и использовать `event.target`, чтобы понять, на каком именно элементе произошел клик.

```javascript
const list = document.querySelector('ul');

list.addEventListener('click', (event) => {
  if (event.target.tagName === 'LI') {
    console.log('Клик по пункту списка:', event.target.textContent);
  }
});
```

---

## 5. Действия браузера по умолчанию

Многие события влекут за собой действия браузера (переход по ссылке, отправка формы). Их можно отменить с помощью `event.preventDefault()`.

```javascript
const link = document.querySelector('a');

link.addEventListener('click', (event) => {
  event.preventDefault(); // ссылка не откроется
  console.log('Переход отменен');
});
```

---

## 6. Основные типы событий

- **Мышь**: `click`, `dblclick`, `mouseover`, `mousedown`.
- **Клавиатура**: `keydown`, `keyup`.
- **Формы**: `submit` (отправка), `input` (изменение поля), `focus`.
- **Документ**: `DOMContentLoaded` (DOM готов).

---

## Лучшие практики

1. **Используйте `addEventListener`**. Это гибко и позволяет вешать несколько функций на одно событие.
2. **Используйте делегирование**. Это экономит память и упрощает работу с динамически добавляемыми элементами.
3. **Не забывайте про `removeEventListener`**. Если вы удаляете элементы со страницы, но на них висят тяжелые обработчики, это может привести к утечкам памяти.
4. **Отменяйте стандартное поведение только когда это нужно**. Пользователи ожидают, что ссылки и формы работают привычным образом.
