# Модуль 2: Типы данных в JavaScript

JavaScript — это язык с **динамической** и **слабой** типизацией. Это означает, что переменная не привязана к конкретному типу данных, а тип определяется значением, которое в ней хранится.

---

## 1. Классификация типов данных

Все типы данных в JS делятся на две большие группы: **Примитивы** и **Объекты**.

### Примитивные типы (7 типов)
Примитивы неизменяемы (immutable) и передаются по значению.

1.  **Number**: Целые числа и числа с плавающей точкой. Включает специальные значения: `Infinity`, `-Infinity` и `NaN` (Not a Number).
2.  **BigInt**: Для целых чисел произвольной длины (добавляется `n` в конце: `100n`).
3.  **String**: Строки. Могут быть в одинарных, двойных или косых (шаблонных) кавычках.
4.  **Boolean**: Логический тип: `true` или `false`.
5.  **null**: Специальное значение, представляющее "ничего" или "пусто".
6.  **undefined**: Значение переменной, которая была объявлена, но не инициализирована.
7.  **Symbol**: Уникальные и неизменяемые идентификаторы.

### Сложный тип: Объект (Object)
Объекты используются для хранения коллекций данных и более сложных сущностей. Передаются по ссылке.
- К объектам также относятся **Массивы (Array)** и **Функции (Function)**.

---

## 2. Определение типа: оператор `typeof`

Оператор `typeof` возвращает строку, указывающую тип операнда.

```javascript
typeof 42;          // "number"
typeof "Hello";     // "string"
typeof true;        // "boolean"
typeof undefined;   // "undefined"
typeof {a: 1};      // "object"
typeof [1, 2];      // "object" (массивы — это объекты!)
typeof function(){};// "function" (хотя функции — это объекты)
typeof null;        // "object" (ОШИБКА В ЯЗЫКЕ, признанная исторически)
```

---

## 3. Приведение типов (Type Coercion)

В JavaScript бывает явное и неявное приведение типов.

### Явное приведение
Используется разработчиком для ясности:
- `String(value)`
- `Number(value)`
- `Boolean(value)`

### Неявное приведение
Происходит автоматически при операциях:
- `5 + "5" // "55"` (число превращается в строку при сложении со строкой)
- `"10" - 2 // 8` (строка превращается в число при вычитании)
- `if (value) ...` (значение приводится к Boolean)

---

## 4. Ложные (Falsy) и Истинные (Truthy) значения

При приведении к логическому типу следующие значения всегда становятся `false`:
- `false`
- `0` (и `-0`, `0n`)
- `""` (пустая строка)
- `null`
- `undefined`
- `NaN`

**Все остальное — `true`** (включая пустые массивы `[]` и объекты `{}`).

---

## 5. Сравнение: `==` против `===`

1.  **Нестрогое сравнение (`==`)**: Приводит типы перед сравнением.
    - `5 == "5" // true`
    - `null == undefined // true`
2.  **Строгое сравнение (`===`)**: Сравнивает и значение, и тип. **Рекомендуется использовать всегда.**
    - `5 === "5" // false`

---

## Резюме
- Используйте примитивы для простых данных.
- Помните, что `null` — это ошибка `typeof`, возвращающая `"object"`.
- Всегда используйте `===`, чтобы избежать странностей неявного приведения.
