# Циклы в JavaScript: Полное руководство

Циклы используются для многократного повторения одного и того же участка кода. Они позволяют автоматизировать однотипные задачи, такие как перебор элементов массива, вывод чисел или обработка данных до достижения определенного условия.

---

## 1. Цикл `while`

Цикл `while` (пока) выполняет код, пока условие истинно (`true`).

```javascript
while (условие) {
  // тело цикла
}
```

**Пример:**
```javascript
let i = 0;
while (i < 3) {
  console.log(i); // Выведет 0, затем 1, затем 2
  i++;
}
```
Если условие изначально ложно, цикл не выполнится ни разу.

---

## 2. Цикл `do...while`

Этот цикл сначала выполняет тело, а затем проверяет условие. Это гарантирует, что код внутри цикла выполнится **хотя бы один раз**.

```javascript
do {
  // тело цикла
} while (условие);
```

**Пример:**
```javascript
let i = 0;
do {
  console.log(i);
  i++;
} while (i < 0); // Условие ложно, но 0 будет выведен в консоль
```

---

## 3. Цикл `for`

Самый популярный и универсальный цикл. Он объединяет в себе начало, условие и шаг.

```javascript
for (начало; условие; шаг) {
  // тело цикла
}
```

- **Начало**: выполняется один раз при входе в цикл (обычно объявление счетчика).
- **Условие**: проверяется перед каждой итерацией. Если `false`, цикл завершается.
- **Шаг**: выполняется после каждой итерации (обновление счетчика).

**Пример:**
```javascript
for (let i = 0; i < 5; i++) {
  console.log(`Итерация: ${i}`);
}
```

---

## 4. Прерывание и пропуск итерации: `break` и `continue`

Иногда нужно остановить цикл раньше времени или пропустить определенный шаг.

### break
Полностью останавливает выполнение цикла.
```javascript
for (let i = 0; i < 10; i++) {
  if (i === 5) break; // Цикл остановится на 5
  console.log(i);
}
```

### continue
Прерывает текущую итерацию и переходит к следующей (к проверке условия или шагу).
```javascript
for (let i = 0; i < 5; i++) {
  if (i === 2) continue; // Число 2 будет пропущено
  console.log(i); // Выведет 0, 1, 3, 4
}
```

---

## 5. Современные циклы: `for...of` и `for...in`

### for...of (Для значений)
Используется для перебора **итерируемых объектов** (массивы, строки).
```javascript
const fruits = ["Яблоко", "Апельсин", "Слива"];

for (let fruit of fruits) {
  console.log(fruit);
}
```

### for...in (Для свойств)
Используется для перебора всех **перечисляемых свойств объекта**.
```javascript
const user = { name: "Ivan", age: 25 };

for (let key in user) {
  console.log(`${key}: ${user[key]}`);
}
```
*Внимание: Не используйте `for...in` для массивов, если порядок элементов важен.*

---

## 6. Вложенные циклы

Вы можете поместить один цикл внутрь другого. Это часто используется для работы с многомерными массивами (матрицами).

```javascript
for (let i = 1; i <= 3; i++) {
  for (let j = 1; j <= 3; j++) {
    console.log(`Координаты: ${i},${j}`);
  }
}
```

---

## 7. Бесконечные циклы

Если условие цикла всегда истинно, он никогда не закончится. Это может привести к зависанию браузера или программы.

```javascript
// ОПАСНО:
// while (true) { ... }
// for (;;) { ... }
```
Всегда проверяйте, что в вашем цикле есть условие выхода или изменение переменной, ведущее к завершению.

---

## Сводная таблица

| Цикл | Когда использовать | Особенности |
| :--- | :--- | :--- |
| **while** | Когда неизвестно точное число итераций | Проверка условия до начала |
| **do...while** | Когда код должен выполниться минимум 1 раз | Проверка условия в конце |
| **for** | Когда известно количество итераций | Вся логика управления в одной строке |
| **for...of** | Для массивов и строк | Работает со значениями |
| **for...in** | Для объектов | Перебирает ключи (свойства) |

---

## Лучшие практики

1. **Используйте `const` в `for...of`**, если не планируете изменять переменную внутри цикла.
2. **Избегайте `for...in` для массивов**, используйте `for...of` или методы массивов (`forEach`, `map`).
3. **Минимизируйте вычисления в условии**. Например, лучше сохранить длину массива в переменную: `let len = arr.length`.
4. **Старайтесь делать циклы простыми**. Если логика внутри цикла слишком сложная, вынесите её в отдельную функцию.
