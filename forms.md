# Работа с формами в JavaScript: Полное руководство

Формы — это основной способ получения данных от пользователя. JavaScript позволяет перехватывать отправку форм, проверять данные на лету и отправлять их на сервер без перезагрузки страницы.

---

## 1. Доступ к формам и элементам

Вы можете найти форму через `querySelector`, но у документов есть и специальная коллекция `document.forms`.

```javascript
const form = document.forms.loginForm; // поиск по атрибуту name
const emailInput = form.elements.email; // поиск элемента внутри формы по name
```

---

## 2. Событие `submit`

Это самое важное событие для форм. Оно срабатывает при попытке отправить форму (клик по кнопке или нажатие Enter).

```javascript
form.addEventListener('submit', (event) => {
  event.preventDefault(); // Останавливаем перезагрузку страницы
  
  console.log('Форма готова к обработке');
});
```

---

## 3. Получение данных: `FormData`

Вместо того чтобы вручную собирать значения каждого поля, используйте удобный объект `FormData`.

```javascript
form.addEventListener('submit', (event) => {
  event.preventDefault();
  
  const formData = new FormData(form);
  
  // Получение конкретного значения
  const email = formData.get('email');
  
  // Превращение всех данных в обычный объект
  const data = Object.fromEntries(formData.entries());
  console.log(data);
});
```

---

## 4. Валидация (Проверка данных)

### Встроенная (HTML5)
Вы можете использовать атрибуты `required`, `type="email"`, `minlength`, `pattern`. Браузер сам покажет ошибку.

### С помощью JavaScript
Для более сложной логики используйте JavaScript.

```javascript
emailInput.addEventListener('input', () => {
  if (emailInput.value.includes('spam')) {
    emailInput.setCustomValidity('Этот email заблокирован');
  } else {
    emailInput.setCustomValidity(''); // Очистка ошибки
  }
});
```

---

## 5. Работа с разными типами полей

- **Текстовые поля**: `input.value`.
- **Чекбоксы**: `input.checked` (возвращает `true/false`).
- **Селекты**: `select.value` (значение выбранной опции).

```javascript
const agree = form.elements.agree.checked;
const city = form.elements.city.value;
```

---

## 6. События полей

- **input**: срабатывает при каждом изменении значения (пока пользователь печатает).
- **change**: срабатывает, когда пользователь закончил ввод и убрал фокус с поля.
- **focus / blur**: когда поле получает или теряет фокус.

---

## Лучшие практики

1. **Всегда используйте `event.preventDefault()`** в обработчике `submit`, если планируете отправлять данные через Fetch.
2. **Используйте `FormData`**. Это стандартный и надежный способ сбора данных, который учитывает даже файлы.
3. **Комбинируйте валидацию**. Используйте атрибуты HTML5 для простых проверок и JavaScript для сложных правил.
4. **Давайте понятную обратную связь**. Если пользователь ошибся, подсветите конкретное поле и напишите, что именно не так.
5. **Блокируйте кнопку отправки**. После клика лучше сделать кнопку `disabled`, чтобы пользователь не отправил одни и те же данные несколько раз, пока идет запрос к серверу.
