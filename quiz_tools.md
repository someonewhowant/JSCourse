# Квиз: Инструменты сборки в JavaScript

Проверьте свои знания об экосистеме современного JavaScript. Ответы в конце.

---

### Вопрос 1
В каком файле хранятся данные о проекте и список всех установленных библиотек?

A) `index.html`  
B) `package.json`  
C) `config.js`  
D) `libraries.txt`  

---

### Вопрос 2
Зачем нужен инструмент **Babel**?

A) Чтобы сжимать картинки  
B) Чтобы переводить современный код JS в старый формат для совместимости  
C) Чтобы удалять вирусы из кода  
D) Чтобы запускать код на сервере  

---

### Вопрос 3
Какую основную задачу выполняет **сборщик** (например, Vite или Webpack)?

A) Проверяет грамматические ошибки в тексте  
B) Объединяет множество модулей в один файл и подготавливает их для браузера  
C) Переводит код на другие языки (английский, русский)  
D) Сохраняет данные в базу данных  

---

### Вопрос 4
Для чего используется инструмент **Prettier**?

A) Для поиска багов в логике программы  
B) Для автоматического форматирования кода (отступы, кавычки) в едином стиле  
C) Для ускорения загрузки сайта  
D) Для защиты от хакеров  

---

### Вопрос 5
Нужно ли отправлять папку `node_modules` в репозиторий Git?

A) Да, обязательно  
B) Только если там мало файлов  
C) Нет, её нужно добавить в `.gitignore`, так как зависимости устанавливаются командой `npm install`  
D) Зависит от версии JavaScript  

---

## Ответы

<details>
<summary>Нажмите, чтобы увидеть ответы</summary>

1. **B) package.json** (Это сердце любого проекта на Node.js или современном JS).
2. **B) Чтобы переводить современный код JS в старый формат...** (Транспиляция).
3. **B) Объединяет множество модулей в один файл...** (Это также называется бандлинг).
4. **B) Для автоматического форматирования кода** (Делает код "красивым" и единообразным).
5. **C) Нет, её нужно добавить в .gitignore** (Эта папка может весить сотни мегабайт, и её легко восстановить по файлу `package.json`).

</details>
