# Обработка ошибок в JavaScript: Полное руководство

Ни один программист не застрахован от ошибок. Ошибки могут возникать из-за неправильного ввода пользователя, проблем с сетью или опечаток в коде. Правильная обработка ошибок позволяет программе не "падать", а корректно реагировать на проблемы.

---

## 1. Конструкция `try...catch`

Это основной инструмент для обработки ошибок.

```javascript
try {
  // Код, который может вызвать ошибку
  console.log(undefinedVariable); 
} catch (error) {
  // Выполнится только если в блоке try произошла ошибка
  console.error("Произошла ошибка!");
  console.log(error.name); // Тип ошибки (например, ReferenceError)
  console.log(error.message); // Сообщение об ошибке
}
```

---

## 2. Блок `finally`

Блок `finally` выполняется **всегда**, независимо от того, была ошибка или нет. Его удобно использовать для очистки ресурсов (например, закрытия соединения или остановки индикатора загрузки).

```javascript
try {
  console.log("Попытка...");
} catch (e) {
  console.log("Ошибка");
} finally {
  console.log("Я выполнюсь в любом случае");
}
```

---

## 3. Генерация собственных ошибок: `throw`

Вы можете сами создавать ошибки, если данные не соответствуют вашим ожиданиям.

```javascript
function checkAge(age) {
  if (age < 0) {
    throw new Error("Возраст не может быть отрицательным");
  }
  return true;
}

try {
  checkAge(-5);
} catch (e) {
  console.log(e.message); // "Возраст не может быть отрицательным"
}
```

---

## 4. Типы встроенных ошибок

- **SyntaxError**: ошибка в синтаксисе кода.
- **ReferenceError**: обращение к несуществующей переменной.
- **TypeError**: операция применена к значению неподходящего типа (например, вызов числа как функции).
- **RangeError**: число вне допустимого диапазона.

---

## 5. Проброс ошибок (Rethrowing)

Если `catch` получил ошибку, которую он не знает как обработать, он может "пробросить" её дальше.

```javascript
try {
  user = JSON.parse(data);
  if (!user.name) {
    throw new SyntaxError("Данные неполны: нет имени");
  }
} catch (e) {
  if (e instanceof SyntaxError) {
    console.log("Ошибка в данных: " + e.message);
  } else {
    throw e; // Пробрасываем ошибку дальше (ReferenceError и т.д.)
  }
}
```

---

## 6. Глобальная обработка ошибок

В браузере можно использовать `window.onerror`, а в Node.js — событие `process.on('uncaughtException')` для перехвата ошибок, которые не были пойманы в `try...catch`. Однако это считается крайней мерой.

---

## Лучшие практики

1. **Не оставляйте `catch` пустым**. Всегда хотя бы логируйте ошибку, иначе отладка станет невозможной.
2. **Используйте конкретные типы ошибок**. Это поможет точнее понимать, что именно пошло не так.
3. **Генерируйте ошибки как можно раньше**. Если функция получает неверные данные, лучше выкинуть ошибку сразу, чем работать с ними.
4. **Не используйте исключения для управления логикой**. Ошибки — это для исключительных ситуаций, а не для обычных `if/else`.
