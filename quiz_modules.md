# Квиз: ES-модули в JavaScript

Проверьте свои знания о том, как организовывать код с помощью модулей. Ответы в конце.

---

### Вопрос 1
Сколько "экспортов по умолчанию" (`export default`) может быть в одном файле?

A) Неограниченно  
B) Не более двух  
C) Только один  
D) Ни одного  

---

### Вопрос 2
Как правильно импортировать именованные сущности `func1` и `func2` из файла `./utils.js`?

A) `import func1, func2 from './utils.js'`  
B) `import { func1, func2 } from './utils.js'`  
C) `import [ func1, func2 ] from './utils.js'`  
D) `import ( func1, func2 ) from './utils.js'`  

---

### Вопрос 3
Какое ключевое слово используется для переименования сущности при импорте?

A) `to`  
B) `is`  
C) `as`  
D) `like`  

---

### Вопрос 4
Что произойдет с переменной, объявленной в модуле без ключевого слова `export`?

A) Она станет глобальной  
B) Она будет доступна во всех файлах проекта  
C) Она будет доступна только внутри этого модуля  
D) Программа выдаст ошибку при запуске  

---

### Вопрос 5
Как браузеру указать, что подключаемый скрипт является модулем?

A) `<script mode="module">`  
B) `<script type="module">`  
C) `<script es6>`  
D) Никак, браузер определит это сам  

---

## Ответы

<details>
<summary>Нажмите, чтобы увидеть ответы</summary>

1. **C) Только один** (Это основное отличие от именованного экспорта).
2. **B) import { func1, func2 } from './utils.js'** (Используются фигурные скобки).
3. **C) as** (Например: `import { longName as short }`).
4. **C) Она будет доступна только внутри этого модуля** (Модули имеют собственную область видимости).
5. **B) <script type="module">** (Это критически важно для работы `import/export` в браузере).

</details>
