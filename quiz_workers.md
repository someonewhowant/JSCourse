# Квиз: Web Workers в JavaScript

Проверьте свои знания о фоновых вычислениях в браузере. Ответы в конце.

---

### Вопрос 1
Какая главная причина использования Web Workers?

A) Чтобы ускорить работу с базой данных  
B) Чтобы выполнять тяжелые вычисления, не блокируя основной поток (интерфейс)  
C) Чтобы получить доступ к файлам на компьютере пользователя  
D) Чтобы заменить язык JavaScript на C++  

---

### Вопрос 2
Имеет ли Web Worker доступ к объекту `document` и элементам страницы?

A) Да, полный доступ  
B) Только для чтения  
C) Нет, воркеры не имеют доступа к DOM  
D) Да, если вызвать метод `worker.enableDOM()`  

---

### Вопрос 3
Какой метод используется для отправки данных между основным скриптом и воркером?

A) `sendData()`  
B) `emit()`  
C) `postMessage()`  
D) `transfer()`  

---

### Вопрос 4
Как остановить воркер из основного скрипта?

A) `worker.stop()`  
B) `worker.kill()`  
C) `worker.terminate()`  
D) `worker.close()`  

---

### Вопрос 5
Что происходит с данными при передаче через `postMessage` по умолчанию?

A) Они передаются по ссылке (быстро)  
B) Они копируются (сериализуются)  
C) Они удаляются из исходного скрипта  
D) Они зашифровываются  

---

## Ответы

<details>
<summary>Нажмите, чтобы увидеть ответы</summary>

1. **B) Чтобы выполнять тяжелые вычисления, не блокируя основной поток** (Это позволяет сохранять плавность интерфейса).
2. **C) Нет, воркеры не имеют доступа к DOM** (Это сделано для безопасности и во избежание конфликтов).
3. **C) postMessage()** (Данные ловятся через событие `onmessage`).
4. **C) worker.terminate()** (Метод `close()` используется внутри самого воркера).
5. **B) Они копируются (сериализуются)** (Поэтому при передаче огромных объемов данных может возникнуть задержка).

</details>
