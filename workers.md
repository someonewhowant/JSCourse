# Web Workers в JavaScript: Полное руководство

**Web Workers** позволяют запускать JavaScript-код в фоновом режиме, отдельно от основного потока выполнения веб-страницы. Это незаменимо для тяжелых вычислений, которые иначе могли бы "заморозить" интерфейс пользователя.

---

## 1. Зачем нужны воркеры?

Браузер выполняет JavaScript в одном потоке (Main Thread). Если вы запустите сложный расчет (например, обработку огромного массива данных), страница перестанет реагировать на клики и прокрутку. Воркер позволяет вынести эту задачу в соседний поток.

---

## 2. Создание воркера

Воркер создается из отдельного файла.

```javascript
// main.js
const myWorker = new Worker('worker.js');
```

---

## 3. Обмен сообщениями

Основной скрипт и воркер общаются через метод `postMessage()` и событие `onmessage`.

### Из основного скрипта в воркер:
```javascript
// main.js
myWorker.postMessage([10, 20]); // Отправляем данные

myWorker.onmessage = (e) => {
  console.log('Результат от воркера:', e.data);
};
```

### Внутри воркера:
```javascript
// worker.js
onmessage = (e) => {
  const result = e.data[0] + e.data[1];
  postMessage(result); // Отправляем результат обратно
};
```

---

## 4. Ограничения воркеров

Воркеры работают в изолированной среде:
- **Нет доступа к DOM**: вы не можете использовать `document` или `window`.
- **Ограниченные API**: доступны таймеры, Fetch API, LocalStorage и некоторые другие, но не все.
- **Копирование данных**: данные, передаваемые через `postMessage`, копируются (сериализуются), а не передаются по ссылке. Это может быть медленно для очень больших объемов данных.

---

## 5. Завершение работы

Если воркер больше не нужен, его следует остановить для экономии ресурсов.

- Из основного скрипта: `myWorker.terminate();`
- Из самого воркера: `self.close();`

---

## 6. Типы воркеров

- **Dedicated Workers**: доступны только тому скрипту, который их создал.
- **Shared Workers**: могут использоваться несколькими окнами или вкладками одного сайта.
- **Service Workers**: особый тип воркеров для кэширования, работы офлайн и пуш-уведомлений.

---

## Лучшие практики

1. **Выносите только тяжелые задачи**. Создание воркера само по себе требует ресурсов, поэтому для простых операций он не нужен.
2. **Обрабатывайте ошибки**. Используйте `myWorker.onerror`, чтобы знать, если в фоновом потоке что-то пошло не так.
3. **Используйте Transferable Objects**. Для передачи очень больших массивов (например, `ArrayBuffer`) можно использовать механизм переноса прав владения, чтобы не копировать данные.
4. **Не храните состояние в воркере**. Воркеры должны быть максимально независимыми и выполнять конкретные функции.
