# Хранение данных в браузере: LocalStorage и SessionStorage

Иногда нужно сохранить данные так, чтобы они не пропадали после перезагрузки страницы или закрытия браузера. Для этого в JavaScript используются объекты `localStorage` и `sessionStorage`.

---

## 1. Различия между LocalStorage и SessionStorage

- **localStorage**: данные сохраняются даже после закрытия браузера и перезагрузки компьютера. У них нет срока годности.
- **sessionStorage**: данные сохраняются только до тех пор, пока открыта текущая вкладка браузера. После закрытия вкладки данные удаляются.

---

## 2. Основные методы

Оба хранилища имеют одинаковый набор методов:

```javascript
// 1. Сохранение данных (только строки!)
localStorage.setItem('theme', 'dark');

// 2. Получение данных
const theme = localStorage.getItem('theme'); // 'dark'

// 3. Удаление конкретного ключа
localStorage.removeItem('theme');

// 4. Полная очистка хранилища
localStorage.clear();

// 5. Количество записей
console.log(localStorage.length);
```

---

## 3. Хранение объектов

**Важно:** Хранилища умеют работать только со строками. Если вы попытаетесь сохранить объект напрямую, он превратится в строку `"[object Object]"`.
Чтобы сохранить объект или массив, используйте JSON.

```javascript
const user = { name: "Ivan", age: 25 };

// Сохранение
localStorage.setItem('user', JSON.stringify(user));

// Получение
const savedUser = JSON.parse(localStorage.getItem('user'));
console.log(savedUser.name); // "Ivan"
```

---

## 4. Событие `storage`

Когда данные в `localStorage` меняются в одной вкладке, во всех остальных вкладках этого же сайта срабатывает событие `storage`. Это позволяет синхронизировать состояние между окнами.

```javascript
window.addEventListener('storage', (event) => {
  console.log('Изменился ключ:', event.key);
  console.log('Новое значение:', event.newValue);
});
```

---

## 5. Ограничения

- **Объем**: обычно около 5-10 МБ на домен.
- **Безопасность**: данные хранятся в открытом виде. Никогда не сохраняйте там пароли, номера карт или токены сессий, если не уверены в безопасности.
- **Синхронность**: операции с хранилищем блокируют основной поток. Если данных очень много, это может замедлить работу сайта.

---

## Лучшие практики

1. **Всегда проверяйте наличие данных**. `getItem` вернет `null`, если ключа не существует.
2. **Используйте `try...catch` при вызове `JSON.parse`**. Если данные в хранилище повреждены, программа может упасть.
3. **Не злоупотребляйте объемом**. Храните только самое необходимое (настройки темы, корзину товаров, черновики).
4. **Очищайте за собой**. Если данные больше не нужны, удаляйте их, чтобы не засорять память пользователя.
