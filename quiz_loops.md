# Квиз: Циклы в JavaScript

Проверьте свои знания о циклах, ответив на следующие вопросы. Ответы приведены в конце файла.

---

### Вопрос 1
Какой цикл гарантирует выполнение тела цикла хотя бы один раз, даже если условие изначально ложно?

A) `while`  
B) `for`  
C) `do...while`  
D) `for...of`  

---

### Вопрос 2
Что будет выведено в консоль в результате выполнения этого кода?

```javascript
for (let i = 0; i < 3; i++) {
  if (i === 1) continue;
  console.log(i);
}
```

A) `0, 1, 2`  
B) `0, 2`  
C) `1, 2`  
D) `0, 1`  

---

### Вопрос 3
Какой из перечисленных способов лучше всего подходит для перебора **значений** в массиве?

A) `for...in`  
B) `while (true)`  
C) `for...of`  
D) `if...else`  

---

### Вопрос 4
Что произойдет при выполнении следующего кода?

```javascript
let count = 5;
while (count > 0) {
  console.log(count);
  // count--; (строка закомментирована)
}
```

A) Выведется число 5 и цикл завершится  
B) Код выдаст ошибку  
C) Ничего не произойдет  
D) Возникнет бесконечный цикл  

---

### Вопрос 5
Как правильно выйти из цикла `for` до того, как условие станет ложным?

A) Использовать `continue`  
B) Использовать `break`  
C) Присвоить счетчику значение `null`  
D) Использовать `return` (вне функции)  

---

## Ответы

<details>
<summary>Нажмите, чтобы увидеть ответы</summary>

1. **C) do...while** (условие проверяется после выполнения тела).
2. **B) 0, 2** (`continue` пропускает текущую итерацию, когда `i` равно 1).
3. **C) for...of** (специально предназначен для перебора значений итерируемых объектов).
4. **D) Возникнет бесконечный цикл** (значение `count` не меняется, поэтому условие `count > 0` всегда истинно).
5. **B) Использовать break** (немедленно прерывает выполнение всего цикла).

</details>
